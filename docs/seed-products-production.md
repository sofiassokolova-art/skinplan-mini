# Заливка продуктов в БД на проде

Если на проде в БД нет продуктов (планы не строятся, пустой каталог), нужно выполнить seed.

## Вариант 1: Локально с продовой БД

1. Убедись, что в `.env` или при запуске задана **продовая** `DATABASE_URL` (из Vercel / хостинга).
2. При необходимости примени миграции:
   ```bash
   npx prisma migrate deploy
   ```
3. Залей бренды и продукты:
   - **100+ продуктов** (как в старой БД) — скрипт на 120 аптечных/маркетплейс продуктов:
     ```bash
     npm run seed:products-120
     ```
   - Меньший набор (~29 продуктов):
     ```bash
     npm run seed:products-full
     ```

Минимальный набор (5 брендов, 5 продуктов):
```bash
npm run seed:products
```

## Вариант 2: Через хостинг (Vercel и т.п.)

Если скрипты не запускаешь с продовой БД с машины, можно:

- Один раз подставить продовый `DATABASE_URL` в `.env` локально и выполнить шаги выше, **либо**
- Добавить одноразовый скрипт/задачу на хостинге, которая при деплое (или вручную) выполняет `npx prisma migrate deploy && npx tsx scripts/seed-products-full.ts` с продовым `DATABASE_URL`.

## Проверка

После seed проверь, что продукты есть:

```bash
npx prisma studio
```

Открой таблицу `products` — должны быть строки с `published: true` / `status: 'published'`.

Или через API (если есть админка): запрос к эндпоинту, который отдаёт список продуктов.
